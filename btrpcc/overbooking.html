Overbooking

Definition

Signature

overbooking(s : set<server>, r : string, x : number)

The overbooking constraint ensures each of the online servers in s overbooks the resource of type r by a factor of at most x. Servers not in the Online state and VMs not in the Running state are ignored.

Usage

This constraints allow the datacenter administrator to control the aggressivity of a non-conservative resource allocation policy. Typically, each VM has a certain amount of virtual CPUs (VCPU) that have to be pinned to the physical CPUs of the allotted server (PCPU). As a VM does not require to consume a full PCPU at every moment, a common practice in industry consists in allocating on average at most 2 VCPU per PCPU. [41] This non-conservative allocation policy can be expressed by the datacenter administrator using one overbooking constraint.

Classification

  • Primary users: datacenter administrator
  • Manipulated elements: Resource allocation
  • Concerns: resource management
